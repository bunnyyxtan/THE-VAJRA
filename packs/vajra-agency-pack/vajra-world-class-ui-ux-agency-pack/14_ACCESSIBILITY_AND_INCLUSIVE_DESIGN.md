# Accessibility and Inclusive Design

Target WCAG 2.2 AA for web and equivalent platform-native accessibility support.

## Color and contrast

- Normal text: at least 4.5:1.
- Large text: at least 3:1.
- Essential controls and graphical objects: applicable non-text contrast.
- Color is never the only signal.
- Check every theme and state.
- Make focus visible and unobscured.
- Do not place critical text on unstable backgrounds.

## Input

- Full keyboard operation on desktop/web.
- VoiceOver on Apple platforms.
- TalkBack on Android.
- Dynamic Type and font scaling.
- Label-in-name compatibility for voice control.
- Large, separated touch targets.
- No gesture-only critical action.
- No keyboard traps.

## Cognitive accessibility

- Stable navigation.
- Consistent language.
- One clear primary action.
- Progressive disclosure.
- Explicit consequences.
- Familiar components.
- Recovery paths.
- Reduced unnecessary motion.
- No unexplained timeouts.
- Preserve entered data.

## Charts

- Text summary.
- Table or data alternative where practical.
- Non-color differentiation.
- Keyboard access for interactive points.
- Clear unit, timeframe, and uncertainty.

## Testing

Automated checks are necessary but insufficient. Include manual assistive-technology testing and representative user testing.
