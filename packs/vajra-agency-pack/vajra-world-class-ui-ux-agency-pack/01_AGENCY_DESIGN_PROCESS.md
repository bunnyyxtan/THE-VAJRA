# Agency Design Process

A world-class interface is not produced by asking an agent to “make it premium.” It is produced through structured discovery, divergent exploration, convergence, prototyping, validation, implementation, and quality control.

## Stage 1 — Discover

Before visual design:

- Audit every current route, object, role, workflow, action, state, permission, data source, and failure condition.
- Interview or observe target users: founders, operators, engineering leads, finance owners, AI-infrastructure engineers, and security reviewers.
- Identify the decisions they make, evidence they need, risks they fear, and information they ignore.
- Review adjacent products, but separate useful interaction patterns from superficial visual trends.
- Gather real examples of incidents, route changes, quality evaluations, provider failures, savings reports, and approvals.
- Document platform context: phone alerts, tablet operations, desktop investigation.

## Stage 2 — Define

Create:

- user groups and jobs-to-be-done,
- task hierarchy,
- information architecture,
- critical journeys,
- decision-risk levels,
- content vocabulary,
- object model,
- status model,
- experience principles,
- non-goals,
- measurable success criteria.

Do not begin high-fidelity design while basic terminology and information architecture are unstable.

## Stage 3 — Develop

Explore several intentionally different design directions.

Each direction must define:

- brand idea,
- visual references,
- typography,
- palette logic,
- layout grammar,
- component character,
- chart language,
- imagery or illustration approach,
- motion behavior,
- light and dark appearance,
- mobile and desktop adaptation.

Do not generate six near-identical dark dashboards with different gradients. Each concept must represent a different strategic hypothesis.

Prototype the most important workflows before building every screen.

## Stage 4 — Deliver

- Build the shared token and component system first.
- Implement complete workflow slices, not isolated screenshots.
- Test with real data extremes and failure states.
- Conduct accessibility, performance, responsive, and platform-convention audits.
- Run usability tests on critical decisions.
- Track design debt and exceptions.
- Release behind controlled gates.
- Measure behavior after release and feed findings back into the design system.

## Required agency artifacts

- Product and UX audit
- User and task model
- Information architecture
- Critical journey maps
- Competitive pattern review
- Brand and visual direction board
- Design principles
- Token system
- Component specifications
- Content standards
- Responsive/adaptive behavior specifications
- Accessibility annotations
- Prototype test report
- Production QA report
- Design decision log
- Post-release metrics plan
