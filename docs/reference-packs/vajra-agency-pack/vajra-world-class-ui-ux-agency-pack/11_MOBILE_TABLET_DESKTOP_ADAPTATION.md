# Mobile, Tablet, and Desktop Adaptation

## Shared meaning, different composition

Share:

- terminology,
- objects,
- statuses,
- semantic colors,
- evidence model,
- permissions,
- action consequences,
- design tokens.

Adapt:

- navigation,
- density,
- layout,
- interaction method,
- depth of information,
- chart complexity,
- confirmation flow.

## Phone

Prioritize:

- health,
- alerts,
- decisions,
- concise evidence,
- spend and savings,
- pause/revert,
- recent activity.

Use:

- native top bars,
- bottom navigation for limited top-level destinations,
- navigation stacks,
- sheets for focused tasks,
- large touch targets,
- direct labels,
- single-column decision flow.

Do not shrink desktop tables or policy editors.

## Tablet

Use:

- list-detail,
- supporting pane,
- split view,
- adaptable inspector,
- keyboard and pointer support,
- rotation and resize continuity.

## Desktop

Use:

- persistent navigation,
- dense tables,
- side-by-side comparison,
- command/search,
- keyboard shortcuts,
- resizable evidence panes,
- advanced filtering,
- saved views,
- multi-window or deep-link workflows where appropriate.

## Continuity

A user should be able to open an alert on mobile, make a safe immediate decision, and continue detailed investigation on desktop without losing the incident, filters, or evidence context.
