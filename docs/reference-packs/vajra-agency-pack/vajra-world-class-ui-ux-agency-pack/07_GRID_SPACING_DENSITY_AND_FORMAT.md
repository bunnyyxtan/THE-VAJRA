# Grid, Spacing, Density, and Page Format

## Spacing system

Use a 4-unit base with an intentional scale:

- 2 — optical micro adjustment
- 4 — tight internal separation
- 8 — related elements
- 12 — compact component spacing
- 16 — standard component spacing
- 24 — groups
- 32 — sections
- 48 — major regions
- 64 — major marketing rhythm

Do not invent values such as 13px, 19px, or 37px without a documented optical reason.

## Layout grids

Recommended starting structures:

- Phone: 4 columns
- Tablet: 8 columns
- Desktop: 12 columns
- Wide analytical workspace: 12 columns with resizable internal panes

Use content constraints, not device labels alone.

## Desktop page shell

1. Global navigation
2. Page identity and scope
3. Health/freshness strip when necessary
4. Primary action area
5. Decision or monitoring summary
6. Main analytical region
7. Supporting evidence or inspector
8. Audit and metadata region

Avoid placing every section in a card. Use page regions, spacing, separators, and surface changes.

## Density modes

VAJRA desktop may support:

- Comfortable — default for general users
- Compact — optional for operators reviewing large tables

Mobile should not expose a desktop-style density toggle.

## Content width

- Prose and explanations: constrained reading width
- Forms: constrained based on field type
- Tables and timelines: may use full workspace width
- Charts: sized around the analytical question, not arbitrary card dimensions

## Responsive behavior

Do not simply stack every desktop card.

At narrower widths:

- preserve the primary decision,
- simplify secondary comparisons,
- convert side inspectors into sheets or routes,
- keep filters available,
- maintain data labels,
- avoid horizontal scrolling except for genuinely tabular data,
- retain selected objects and user state.
