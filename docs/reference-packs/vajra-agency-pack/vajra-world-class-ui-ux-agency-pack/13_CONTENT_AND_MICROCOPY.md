# Content and Microcopy

VAJRA’s writing should be concise, precise, calm, and accountable.

## Voice

- factual,
- direct,
- technically credible,
- non-promotional inside the product,
- explicit about uncertainty,
- explicit about consequences,
- helpful during recovery.

## Avoid

- AI-powered
- revolutionary
- seamless
- magical
- smart optimization
- next-generation
- one-click everything
- vague “Something went wrong”
- unexplained provider, model, wallet, RPC, or policy codes

## Decision copy

A proposed action should state:

- what will change,
- why,
- estimated effect,
- evidence confidence,
- policy result,
- risk,
- reversibility,
- required user action.

## Error copy format

1. What happened
2. What was preserved
3. What did not happen
4. What the user can do
5. Where to get more detail

Example:

“VAJRA could not apply the route change because the provider health check timed out. The current route is still active and no configuration was changed. Try again, review provider status, or keep the proposal for later.”

## Labels

Prefer:

- `Approve route change`
- `Keep current route`
- `Pause automatic switching`
- `Review evaluation evidence`
- `Revert to previous route`

Avoid vague `Continue`, `Submit`, `Yes`, or `No` for consequential actions.
