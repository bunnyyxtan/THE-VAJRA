# No AI-Slop Standard

“AI slop” is not a formal usability category. In this project it means visually generic, context-free, overdecorated, inconsistent, or unverified output produced because it looks impressive in a screenshot rather than because it supports VAJRA’s tasks.

## Automatic rejection patterns

Reject a design when it contains several of these without strong product justification:

- dark navy background plus purple-blue gradient glow,
- giant generic headline with vague claims,
- glowing orb or abstract 3D object,
- every section inside a rounded floating card,
- excessive glassmorphism and backdrop blur,
- pills used for every label,
- random neon borders,
- many unrelated gradients,
- oversized empty spacing that hides weak hierarchy,
- dashboards made entirely of identical metric cards,
- decorative charts with fake numbers,
- arbitrary serif headings paired with generic sans-serif body text,
- excessive icon-only controls,
- motion on every scroll or hover,
- repeating “AI-powered,” “next generation,” “revolutionary,” or “seamless” copy,
- one design resized unchanged across phone and desktop,
- nonfunctional prototypes replacing real workflows,
- invented data or claims.

## Visual rationale rule

Every major design choice must answer at least one question:

- Does it clarify hierarchy?
- Does it reveal state?
- Does it improve comparison?
- Does it reduce error?
- Does it improve orientation?
- Does it strengthen VAJRA’s original identity?
- Does it make a high-risk decision safer?
- Does it improve accessibility?
- Does it improve perceived or actual performance?

If the answer is no, remove it.

## Originality test

Before approval, compare the design against:

- popular AI landing-page patterns,
- generic shadcn dashboard templates,
- common Dribbble crypto dashboards,
- current SaaS boilerplates.

The design must remain recognizably VAJRA when:

- the logo is removed,
- brand name is hidden,
- colors are temporarily desaturated.

Identity should survive through structure, terminology, data treatment, decision anatomy, and interaction behavior—not only a logo and accent color.

## Human-review requirement

AI may accelerate exploration and production, but a human must review:

- visual hierarchy,
- originality,
- cultural implications,
- data meaning,
- high-risk actions,
- copy,
- accessibility,
- responsive behavior,
- motion,
- final implementation quality.
